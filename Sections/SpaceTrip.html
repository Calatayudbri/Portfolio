SPACE TRIP

Restricted Input Academic Project

Space Trip is an academic project developed in Unity with a unique constraint: only gamepad joysticks can be used for interaction. This limitation created an interesting challenge in designing intuitive controls for space navigation, asteroid avoidance, and survival gameplay mechanics.

🚀

Restricted Input System

The core challenge of this project was creating engaging gameplay using only the left and right joysticks of a gamepad. No face buttons, triggers, or D-pad interactions were allowed, requiring creative solutions for all game mechanics.

Joystick Mapping Strategy

The Left Joystick controls ship acceleration and deceleration using vertical input, while the Right Joystick manages 3D rotation around all axes, providing full spatial control.

This design allows for intuitive space navigation while maintaining the challenge of the input restriction. Players can smoothly accelerate forward or backward while independently controlling their ship's orientation.

// Left stick vertical: Speed control
if (Input.GetAxis("Left_Stick_v") == -1) {
    if (speed_ <= max_speed_ && speed_ >= min_speed_) {
        speed_ += 0.1f;
        particleSystem.Play();
    }
}

// Right stick: 3D rotation control
if (Input.GetAxis("Right_Stick_h") == 1) {
    transform.Rotate(Vector3.up * rootSpeed_left * Time.deltaTime);
}

Left Joystick - Propulsion

Vertical movement controls ship speed. Pull down to accelerate forward, push up to brake or reverse. This creates an intuitive "throttle" feeling similar to real spacecraft controls.

2

Right Joystick - Attitude Control

Full 3D rotation mapping: horizontal for yaw, vertical for pitch. This allows players to orient their ship in any direction while maintaining forward momentum.

Menu Navigation

Survival Gameplay Mechanics

The game focuses on survival through asteroid fields, with dynamic difficulty scaling and immersive feedback systems that respond to player performance and create escalating tension.

Asteroid Generation

The AsteroidSpawner creates threats that home toward the player's position at spawn time:

  • Predictive targeting: Asteroids aim for player's location when created
  • Speed variation: Random speeds between 3.5f - 9.5f units
  • Lifespan control: Auto-destruction prevents memory leaks
  • Rotational movement: Visual spinning for realism

Dynamic Camera System

Camera responds to player actions creating immersion:

  • Speed-based FOV: Field of view narrows with acceleration
  • Following behavior: Smooth tracking with configurable distance
  • Performance integration: FOV changes reflect speed dynamically
  • Depth perception: Enhanced sense of speed and danger

Survival Timer

Core scoring mechanism tracks player performance:

  • Real-time tracking: Precision timer with milliseconds
  • Best time records: Persistent high score system
  • Death state handling: Clean timer stop on collision
  • Visual feedback: UI color changes to indicate danger
// Asteroid homing behavior
private void Start() {
    GameObject playerShip = GameObject.FindGameObjectWithTag("Player");
    if (playerShip != null) {
        playerposition = playerShip.transform.position;
    }
    direction_vector = (playerposition - transform.position).normalized;
    Destroy(gameObject, lifeSpan);
}
Asteroid targeting system implementation

Intelligent Threat Generation

Asteroids use predictive targeting, calculating the player's position at spawn time rather than continuously tracking. This creates strategic gameplay where players must anticipate incoming threats.

The system balances challenge and fairness - threats are dangerous but avoidable through skillful maneuvering and speed management.

User Interface Systems

All UI systems maintain consistency with the input restrictions while providing essential feedback for navigation, performance tracking, and game state management.

Menu Navigation System

The main menu system adapts to joystick-only input using DPAD_V axis mapping for vertical navigation and submit/cancel buttons for selection.

Visual feedback includes button scaling, color changes, and smooth transitions to clearly indicate the current selection without relying on mouse hover states.

// Menu navigation with gamepad
float verticalInput = Input.GetAxis("DPAD_v");

if (verticalInput > 0.5f) {
    NavigateUp();
} else if (verticalInput < -0.5f) {
    NavigateDown();
}

if (Input.GetButtonDown("Submit")) {
    SelectCurrentButton();
}

Technical Implementation

The project demonstrates several technical solutions including performance optimization, persistent data storage, and adaptive visual effects that enhance the core gameplay experience.

Performance Optimization

Key optimization strategies implemented:

  • Object pooling: Efficient asteroid lifecycle management
  • Automatic cleanup: Timed destruction prevents buildup
  • Smart collision: Tag-based collision detection
  • Particle management: Conditional particle system activation

Data Persistence

Save system features:

  • Best time tracking: PlayerPrefs integration
  • Cross-session persistence: Records survive game restarts
  • Formatted display: MM:SS:MS time presentation
  • Record validation: Only saves improved times

Visual Effects System

Immersive feedback mechanisms:

  • Engine particles: Visual thruster effects
  • Explosion effects: Collision impact feedback
  • Dynamic FOV: Speed-responsive camera effects
  • UI state changes: Color coding for game states
