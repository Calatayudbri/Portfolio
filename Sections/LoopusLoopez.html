Paella Defender

A frantic top-down action game where players defend their paella from waves of enemies. I developed the core systems including dynamic audio management, sophisticated scoring mechanics, and modular developer settings. Built entirely in Unreal Engine 5 using C++ and Blueprint systems.

Advanced Scoring System

A sophisticated multi-layered scoring system that rewards skill, timing, and creative play through dynamic multipliers, combo systems, and performance-based bonuses. Designed to encourage aggressive and skillful gameplay.

Final Score = Base + Base × (Bounces×1.5 + Combo×Multi + Velocity×Multi + Explosion×3.0)

Multi-Factor Calculation

The scoring system evaluates multiple performance factors for each enemy elimination. Base scores are modified by bounce count, combo multipliers, projectile velocity, and special elimination types like explosions.

Every time a combo and score are updated, we use delegates to notify it and update the UI

1

Death Event Processing

When an enemy dies, the system collects performance data including projectile speed, bounce count, and elimination method through the NotifyDeath() function.

2

Velocity Bonus Calculation

Uses configurable curves to map projectile velocity to score multipliers, rewarding players for fast, precise shots and encouraging dynamic gameplay.

3

Combo System Management

Tracks consecutive eliminations with timeout mechanics. Combo multipliers increase with chain length, but reset if players take too long between kills.

4

Level Progression

Accumulated score triggers level-ups with scaling upgrade costs, integrating the scoring system directly with character progression and upgrade selection.

Combo System

Dynamic combo tracking with configurable parameters:

  • Combo Threshold: Minimum kills to activate multipliers
  • Multiplier Scaling: Linear increase per combo level
  • Reset Conditions: Timeout

Special Modifiers

Bonus multipliers for skillful play:

  • Bounce Multiplier: Multiplier per ricochet before hitting target
  • Velocity Bonus: Curve-based speed rewards
  • Explosion Bonus: Multiplier for area-of-effect kills
  • Precision Rewards: Higher scores for direct hits

Data Structure

The FDeathParams struct contains:

  • BulletSpeed: Projectile velocity at impact
  • BulletBaseScore: Base score value
  • numBounces: Ricochet count before elimination
  • bIsExplosionDeath: Area damage bonus flag
  • bIsMeleeDeath: Close combat penalty flag

Upgrade Integration

The scoring system directly drives character progression through level-based upgrades. As players accumulate score, they unlock upgrade choices that modify gameplay mechanics, creating a feedback loop between performance and power growth.

Upgrade costs scale exponentially using configurable base values and level multipliers, ensuring progression remains challenging throughout the game.

void AScoreManager::CheckLevelUpgrade()
{
    if(CurrentLevelScore >= NextLevelScore)
    {
        CurrentLevel++;
        float RemainingScore = CurrentLevelScore - NextLevelScore;
        
        NextLevelScore = BaseUpgradeCost + 
            (BaseUpgradeCost * (CurrentLevel * UpgradeCostMultiplier));
        
        CurrentLevelScore = RemainingScore;
        
        if(UM) {
            UM->ShowsUpgradeScreen();
        }
    }
}

Modular Developer Settings

A centralized configuration system using Unreal Engine's Developer Settings framework, allowing designers to fine-tune game balance parameters without code changes. All scoring, upgrade, and gameplay values are exposed through a clean, organized interface.

Centralized Configuration

The ULLDeveloperSettings class extends Unreal's Developer Settings system, creating a global configuration hub accessible from any part of the codebase using GetDefault<ULLDeveloperSettings>().

This approach eliminates hardcoded values, enables rapid iteration, and provides designers with direct control over game balance without programmer intervention.

UCLASS(Config = "Game", defaultConfig, 
       meta = (DisplayName = "Lupus lopez global params"))
class LUPUSLOPEZ_API ULLDeveloperSettings : public UDeveloperSettings
{
    GENERATED_BODY()
public:
    UPROPERTY(Config, EditAnywhere, Category = "Score Params")
    FScoreParams ScoreParams;
};
1

Settings Structure

The FScoreParams struct organizes related parameters into logical groups, making the editor interface clean and intuitive for designers.

2

Runtime Access

Game systems access settings through a global singleton pattern, ensuring consistent values across all systems and enabling real-time parameter updates during development.

3

Configuration Persistence

Settings are automatically saved to configuration files and synchronized across team members, ensuring consistent game balance across development builds.

Score Parameters

  • BounceMultiplier: Score bonus per projectile ricochet
  • ComboMultiplier: Multiplier increase per combo level
  • VelocityCurveToScore: Curve asset for speed bonuses
  • ComboThreshold: Minimum eliminations for combo activation
  • ComboTimeWindow: Maximum delay between combo hits
  • ExplosionScoreMultiplier: Area damage bonus multiplier

Upgrade Configuration

  • BaseUpgradeCost: Initial score requirement for first upgrade
  • UpgradeCostByLevelMultiplier: Exponential scaling factor
  • Category Organization: Logical parameter grouping
  • Editor Integration: Native Unreal Engine interface

System Benefits

float ScoreModifier = (ScoreParams.BounceMultiplier * DeathParams.numBounces) +

Implementation Pattern

Systems initialize settings references during startup and use them throughout gameplay calculations. This pattern ensures consistency while maintaining performance through cached references.

The configuration system integrates seamlessly with Unreal's project settings, appearing in the editor alongside other engine settings for easy access.
