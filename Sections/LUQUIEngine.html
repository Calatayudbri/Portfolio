LUQUI ENGINE

Custom OpenGL Graphics Engine

A modern 3D graphics engine built from scratch using OpenGL, C++, and modern rendering techniques. Features include asynchronous model loading, deferred rendering pipeline, dynamic lighting with shadow mapping, and comprehensive material systems.

Model Loading System

Advanced 3D model loading system with asynchronous processing, multi-threading support, and efficient memory management. Supports various formats through Assimp integration and features smart caching mechanisms.

Asynchronous Loading Architecture

The model loading system utilizes a custom JobSystem for asynchronous processing, preventing main thread blocking during heavy I/O operations.

Models are loaded in two phases: data processing on worker threads and OpenGL resource creation on the main thread, ensuring thread safety while maximizing performance.

Model::Model(const std::string& path, JobSystem& jobSystem) : asyncMode(true) { loadModelDataAsync(path, jobSystem); } void Model::loadModelDataAsync(const std::string& path, JobSystem& jobSystem) { jobSystem.addJob(JobSystem::JobType::General, JobSystem::JobPriority::High, [this, path]() { // Process model data on worker thread Assimp::Importer importer; const aiScene* scene = importer.ReadFile(path, aiProcess_Triangulate | aiProcess_FlipUVs); processNode(scene->mRootNode, scene); dataLoaded = true; }); }
1

File Processing

Models are loaded using Assimp library with optimized flags for triangulation and UV coordinate processing.

2

Mesh Data Extraction

Vertex data, indices, and material information are extracted and stored in intermediate structures for later OpenGL buffer creation.

3

Texture Management

Smart texture caching system prevents duplicate loading of textures, optimizing memory usage across multiple models.

4

OpenGL Resource Creation

VAOs, VBOs, and textures are created on the main thread once data processing is complete, ensuring thread safety.

Mesh Structure

Each mesh contains optimized vertex data:

  • Position: 3D world coordinates
  • Normal: Surface normal vectors
  • Texture Coordinates: UV mapping
  • Material Data: Diffuse, specular, normal maps

Memory Optimization

Smart memory management features:

  • RAII-based resource management
  • Move semantics for efficient transfers
  • Automatic cleanup on destruction
  • Shared texture resource pooling

Rendering Pipeline

Flexible rendering architecture supporting both forward and deferred rendering pipelines. The deferred renderer optimizes performance for scenes with multiple lights while maintaining visual quality.

Forward Rendering

Advantages:

  • Simple implementation
  • Transparent object support
  • Lower memory usage
  • MSAA compatibility

Best for: Scenes with few lights, mobile rendering

Deferred Rendering

Advantages:

  • Constant lighting cost
  • Many lights support
  • Decoupled geometry/lighting
  • Advanced effects ready

Best for: Complex scenes with many dynamic lights

class DeferredRenderer { private: GLuint gBuffer; // G-Buffer framebuffer GLuint gPosition; // Position texture GLuint gNormal; // Normal texture GLuint gAlbedoSpec; // Albedo + Specular GLuint rboDepth; // Depth renderbuffer public: void GeometryPass(ECSManager& ecsManager, Program& gBufferProgram); void LightingPass(ECSManager& ecsManager, Program& lightProgram); void FinalPass(Program& finalProgram); };

G-Buffer Architecture

The deferred renderer uses a multi-target framebuffer (G-Buffer) to store geometry information:

Position Buffer (RGBA16F): World-space positions with depth

Normal Buffer (RGBA16F): World-space normals

Albedo-Spec Buffer (RGBA16F): Diffuse color and specular intensity

1

Geometry Pass

All opaque objects are rendered to the G-Buffer, storing position, normal, and material data for later use.

2

Lighting Pass

Screen-space lighting calculations are performed using G-Buffer data with additive blending for multiple lights.

3

Final Pass

Results are composed and tone-mapped to the screen, with optional post-processing effects applied.

Lighting & Shadow System

Comprehensive lighting system supporting multiple light types with advanced shadow mapping techniques. Features directional, point, and spot lights with cascaded and cube shadow maps.

Directional Lights

  • Orthographic shadow projection
  • Camera-frustum aligned shadows
  • Infinite light sources (sun/moon)
  • Efficient for outdoor scenes

Point Lights

  • Omnidirectional shadow casting
  • Cube shadow maps (6 faces)
  • Distance-based attenuation
  • Perfect for indoor lighting

Spot Lights

  • Perspective shadow projection
  • Configurable cone angles
  • Smooth falloff at edges
  • Great for focused lighting

Shadow Mapping Implementation

The shadow system uses multiple techniques based on light type:

Standard Shadow Maps: For directional and spot lights using depth textures

Cube Shadow Maps: For point lights using cube map depth textures

Dynamic Resolution: 1024x1024 shadow maps with configurable quality settings

void InitShadowForLight(LightComponent& light) { const unsigned int SHADOW_WIDTH = 1024, SHADOW_HEIGHT = 1024; if (light.type == LightType::Point) { // Create cube shadow map for omnidirectional shadows glGenTextures(1, &light.cubeDepthMap); glBindTexture(GL_TEXTURE_CUBE_MAP, light.cubeDepthMap); for (unsigned int i = 0; i < 6; ++i) { glTexImage2D(GL_TEXTURE_CUBE_MAP_POSITIVE_X + i, 0, GL_DEPTH_COMPONENT, SHADOW_WIDTH, SHADOW_HEIGHT, 0, GL_DEPTH_COMPONENT, GL_FLOAT, NULL); } } else { // Create standard depth map for directional/spot lights glGenTextures(1, &light.depthMap); glBindTexture(GL_TEXTURE_2D, light.depthMap); glTexImage2D(GL_TEXTURE_2D, 0, GL_DEPTH_COMPONENT, SHADOW_WIDTH, SHADOW_HEIGHT, 0, GL_DEPTH_COMPONENT, GL_FLOAT, NULL); } }
1

Shadow Map Generation

Depth maps are rendered from each light's perspective, storing distance information for shadow calculations.

2

Light Space Transformation

World positions are transformed to light space using appropriate projection matrices for each light type.

3

Shadow Testing

Fragment depths are compared against shadow map values to determine if surfaces are in shadow.

4

Soft Shadow Filtering

PCF (Percentage Closer Filtering) is used to create smooth shadow edges and reduce aliasing artifacts.

Skybox System

Environmental rendering system using cube maps to create infinite distance backgrounds. Integrated seamlessly with the rendering pipeline for realistic sky and environment representation.

class Skybox { private: unsigned int textureID; // Cube map texture unsigned int VAO, VBO; // Geometry buffers public: Skybox(const std::vector& faces); void Draw(Program& skyboxProgram, const glm::mat4& view, glm::mat4& projection); private: void LoadCubemap(const std::vector& faces); void SetupCube(); }; void Skybox::Draw(Program& skyboxProgram, const glm::mat4& view, glm::mat4& projection) { glDepthFunc(GL_LEQUAL); // Render at max depth // Remove translation from view matrix glm::mat4 viewNoTranslation = glm::mat4(glm::mat3(view)); skyboxProgram.setmat4("view", viewNoTranslation); skyboxProgram.setmat4("projection", projection); glBindTexture(GL_TEXTURE_CUBE_MAP, textureID); glDrawArrays(GL_TRIANGLES, 0, 36); }

Cube Map Implementation

The skybox system uses OpenGL cube maps with six texture faces representing the complete environment around the viewer.

Depth Optimization: Skybox is rendered with maximum depth value and GL_LEQUAL depth testing

View Matrix Handling: Translation is removed from the view matrix to maintain infinite distance illusion

1

Cube Map Loading

Six texture images are loaded and bound to cube map faces: +X, -X, +Y, -Y, +Z, -Z representing right, left, top, bottom, front, back.

2

Geometry Setup

A unit cube is generated with vertices optimized for cube map sampling, ensuring proper texture coordinate interpolation.

3

Rendering Integration

Skybox is rendered after opaque geometry but before transparent objects, with depth writing disabled.

Texture Configuration

  • Format: RGB for color data
  • Filtering: Linear interpolation
  • Wrapping: Clamp to edge
  • Mipmaps: Not required for skyboxes

Performance Features

  • Early depth rejection optimization
  • Minimal vertex processing
  • Efficient cube map sampling
  • Single draw call rendering
